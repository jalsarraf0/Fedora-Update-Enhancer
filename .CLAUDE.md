## What It Is
Single Bash script (`elegant-updater.sh`, 695 lines) for high-performance unattended
Fedora DNF5 upgrades. Adaptive parallelism based on CPU load + link speed.
Installed on this host as `sudo update` (symlink at `~/scripts/update`).

## Invocation
```bash
sudo update                           # canonical on this host
sudo bash elegant-updater.sh          # direct
```

## Key Internal Functions
| Function | Purpose |
|---|---|
| `calc_jobs_from_load()` | awk-based: maps CPU load ratio → job count (min/max range) |
| `detect_fastest_link_speed_mbps()` | reads `/sys/class/net/*/speed`, skips virtual NICs |
| `clamp_int()` | clamps value between lo/hi |
| `as_positive_int()` | validates positive integer or returns fallback |

## Output Helpers (ANSI, terminal-guarded by `[[ -t 1 ]]`)
`hr()`, `ttl()`, `ok()`, `warn()`, `err()`, `note()`
Colours: `C_HL` (cyan), `C_OK` (green), `C_WARN` (yellow), `C_ERR` (red), `C_NEON` (bright cyan).
In non-TTY contexts (cron, pipe) all ANSI codes are empty strings — safe to pipe.

## DNF5 Config Path Detection
```bash
DNF=${DNF:-/usr/bin/dnf5}
# Auto-detects: /etc/dnf/dnf5.conf (dnf5) → fallback to /etc/dnf/dnf.conf
```

## Tunables (environment variables)
| Variable | Purpose |
|---|---|
| `JOBS` | Parallel jobs for dnf5 (auto-calculated if unset) |
| `MAX_PARALLEL_DOWNLOADS` | Cap on concurrent downloads |
| `DNF_PARALLEL_CAP` | Hard cap on DNF parallelism |
| `FASTESTMIRROR` | Enable fastest mirror selection |
| `PREFER_MIRRORS` | Mirror preference list |
| `RUN_UPDATE_SWEEP` | Run a post-update sweep (cleanup, orphan check) |
| `SHOW_REPO_LIST` | Print repo list during run |
| `DNF` | Override DNF binary path (default: `/usr/bin/dnf5`) |
| `DNF_CONF` | Override DNF config path |

## Virtual NIC Skip List (in detect_fastest_link_speed_mbps)
`lo, docker*, veth*, br-*, virbr*, tun*, tap*, wg*, zt*` are excluded.

## Validation
```bash
shellcheck elegant-updater.sh
# Real test requires Fedora 43 host with dnf5
```

## Hard Rules
- Use `dnf5` only — no `dnf4` fallback paths.
- No interactive prompts — this is unattended.
- Always use the defined output helpers; never raw `echo` for user-facing output.
- `[[ -t 1 ]]` TTY guard must remain on all ANSI colour setup.
- Idempotent: safe to run multiple times in sequence.
- Never modify the `calc_jobs_from_load` awk formula without benchmarking on real hardware.
