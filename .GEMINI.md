# .GEMINI.md - Fedora-Update-Enhancer

This project consists of a single Bash script, `elegant-updater.sh`, designed to be a high-performance, unattended updater for Fedora systems using `dnf5`.

### Functionality
The script is intended to be installed as `sudo update` and provides several enhancements over a standard `dnf5 upgrade`:
- **Adaptive Parallelism:** Automatically tunes the number of parallel downloads based on CPU cores, current system load, and network link speed.
- **DNF5 Configuration:** Dynamically modifies `dnf.conf` or `dnf5.conf` to optimize settings like `fastestmirror`, `deltarpm`, and `max_parallel_downloads`.
- **Repository Failover:** If a repository is unavailable, the script can automatically retry the update with the failing repository temporarily disabled.
- **Automated Cleanup:** Runs `dnf5 autoremove` and cleans up old packages after the update is complete.

### Execution
The script must be run with `sudo`.
- **As installed command:**
  ```bash
  sudo update
  ```
- **Directly from the repository:**
  ```bash
  sudo ./elegant-updater.sh
  ```

### Installation
To make the script available as the `update` command system-wide:
```bash
sudo install -m 0755 elegant-updater.sh /usr/local/bin/update
```

### Configuration
The script's behavior can be heavily customized using environment variables. For example:
- `MAX_PARALLEL_DOWNLOADS=20`: Manually set the number of parallel downloads.
- `PREFER_MIRRORS=0`: Disable the feature that prefers mirror lists over base URLs.
- `JOBS=8`: Manually set the number of CPU worker jobs.

### Validation
- `shellcheck elegant-updater.sh`: Check for common scripting errors.
- `bash -n elegant-updater.sh`: Validate the script's syntax without executing it.
